# APM導入 Phase 4: CI連携の検討

## 背景
APMの検証をCIへ組み込むことで、設定やlockfileの不整合を早期に検知できる可能性があります。一方で、外部依存取得や環境差分によりCIが不安定になる可能性もあります。

## スコープ
- `apm install`、`apm audit`、CI向けオプションの実行可否確認
- GitHub Actions追加の要否判断
- warning運用または必須チェック化の判断

## タスク
- [ ] ローカルで利用しているAPM検証コマンドを整理する
- [ ] CIで必要な依存関係と実行時間を確認する
- [ ] 外部依存取得失敗時の扱いを決める
- [ ] GitHub Actionsを追加する場合は最小構成で作成する
- [ ] `docs/templates/change-log.md` に変更を記録する

## 完了条件
- [ ] CIへ組み込むかどうか判断済みである
- [ ] 組み込む場合、失敗条件とwarning条件が明確である
- [ ] 組み込まない場合、手動検証コマンドがREADME等に明記されている

## 依存関係
- Phase 1の検証コマンドが安定していること
- Phase 3の外部依存方針が整理されていること
